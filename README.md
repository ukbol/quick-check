# quick-check — BOLD & DToL Species Lookup

A static, single-page tool for looking up UK species against the UKBOL / BOLD / DToL
dataset. Paste species names (one per line) and get coverage, BAGS grade, status, and
DToL priority. Synonyms and close (fuzzy) spellings are matched automatically.

## How it works

The page runs entirely in the browser — no server required.

- **`build.py`** turns the dated source TSV into two files: **`data.json`** (a compact,
  columnar, dictionary-encoded payload — only the displayed columns are kept) and
  **`data.meta.json`** (a tiny manifest with a `version`, byte size and row count).
- **`worker.js`** is a Web Worker that streams + parses `data.json` and runs all
  searching **off the main thread**, so the page never freezes while loading and stays
  responsive on mobile. It reports download progress to the page.
- **`service-worker.js`** + **`manifest.json`** make the app an installable, **offline**
  PWA: after the first visit the app shell and dataset are cached, so later visits load
  instantly and work with no network. "Add to Home Screen" installs it like an app.

### Performance notes

`build.py` drops the columns the page never shows (`organism_key`, `taxon_version_key`,
`kingdom`, `phylum_division`, `class`) and dictionary-encodes low-cardinality columns,
which roughly halves the raw parse size and CPU cost. The transfer itself is gzip ~1.6 MB:
**GitHub Pages serves gzip but not brotli**, so we can't shrink the bytes much further —
the speed-ups come from parsing off-thread and from caching every load after the first.
(A host that serves brotli — Cloudflare Pages, Netlify, Vercel — would cut the first-load
transfer ~25% with no code change.)

## Updating the data

1. Drop the new dated source file (e.g. `2026_06_06_bold_dtol.txt`) into the repo root.
2. Regenerate the data files:
   ```sh
   python3 build.py
   ```
   It auto-selects the newest `*_bold_dtol.txt` (or pass a path:
   `python3 build.py path/to/file.txt`) and writes `data.json` + `data.meta.json`.
3. Commit the source TSV, `data.json`, and `data.meta.json`.

`index.html` always loads `data.json`, so no code edit is needed on each update. The new
`version` in `data.meta.json` tells the service worker the data changed: clients that
already have the old data cached show a "New data available — Refresh" banner.

If you change `index.html`, `worker.js`, or `service-worker.js`, bump `CACHE_VERSION` at
the top of `service-worker.js` so clients pick up the new shell.

## Branding / logo

The header expects the UKBOL logo at `assets/images/ukbol-text-logo-clear.png`. If the
file is absent it falls back to a "UKBOL" text wordmark automatically. Drop the real PNG
in that path to display it.

The PWA icons in `assets/icons/` are generated by `generate_icons.py` (a DNA-barcode motif
in the UKBOL palette, no image libraries required). Re-run it only if you want to change
them; the PNGs are committed.

## Search behaviour

- **Exact** (case-insensitive) match on taxon name or any synonym.
- **Fuzzy fallback** when there's no exact match — a "Did you mean?" list of approximate
  matches by edit distance (a few different letters) and by species epithet (same epithet,
  different genus). Click a suggestion to view its full record.
